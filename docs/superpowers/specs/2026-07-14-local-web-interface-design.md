# 本地网页界面设计说明

## 目标

在不改变现有命令行自动文件流程的前提下，为简历与 JD 精准匹配 Agent 增加一个仅在本机访问的 Gradio 网页界面。

保留两条独立入口：

- `python src/main.py`：继续自动读取项目根目录的 `resume.txt`、`jd.txt`，执行分析并生成 `result.txt`。
- `python src/web.py`：启动本地网页，自动加载上述两个输入文件，允许临时编辑、显式保存和交互式分析。

## 方案选择

采用独立 Gradio 页面，而不是标准库 HTTP 服务、Flask 或 Sites 前端工程。

- Gradio 能直接调用现有 Python Agent，减少重复接口层。
- 单独的 `src/web.py` 不会增加 `src/main.py` 的职责，也不会使命令行流程依赖网页组件。
- 项目没有 `.openai/hosting.json` 或现有 JavaScript 前端结构，并且用户要求仅本地运行，因此不创建独立 Sites 工程，也不发布外网地址。

## 文件职责

### `src/web.py`

负责本地网页相关功能：

- 定位项目根目录。
- 读取 `resume.txt` 和 `jd.txt` 作为页面初始值。
- 显式保存网页文本到两个输入文件。
- 调用 `ResumeAgent.run()` 执行分析。
- 每次成功分析后调用现有 `write_result()` 更新项目根目录的 `result.txt`。
- 将结构化结果转换为适合页面展示的评分、技能、硬性条件、STAR、关键词、JSON 和总结视图。
- 构建 Gradio Blocks 页面并启动本地服务。

Gradio 只在构建或启动页面时导入。纯文件及分析辅助函数可在没有 Gradio 的测试环境中导入和测试。

### 现有业务文件

- `src/main.py` 保持现有自动文件入口和行为。
- `src/agent.py` 继续负责串行编排，不加入网页组件。
- `src/tools.py` 和 `src/model_client.py` 不包含网页逻辑。

### 配置与文档

- `environment.yml` 增加 `gradio`。
- `README.md` 分别说明命令行入口和网页入口。

## 输入与保存语义

网页启动时读取项目根目录输入文件：

- 文件存在且是 UTF-8：内容自动填入对应文本框。
- 文件缺失、为空或无法读取：文本框保持可编辑，并在状态区显示明确提示。

页面编辑与磁盘文件相互隔离：

- 点击“开始精准分析”使用页面当前文本，不覆盖 `resume.txt` 或 `jd.txt`。
- 点击“重新加载文件”放弃页面尚未保存的修改，再次读取磁盘内容。
- 只有点击“保存到文件”才写回两个输入文件。
- 保存采用同目录临时文件加替换的方式，两个文件均使用 UTF-8。
- 保存前校验两段内容非空；校验失败时不修改任何输入文件。

## 分析数据流

1. 用户打开页面，系统自动加载输入文件。
2. 用户可以编辑文本，但不会自动写回磁盘。
3. 用户点击“开始精准分析”。
4. 页面校验简历和 JD 均非空。
5. 调用现有 `ResumeAgent.run(resume, jd)`，其内部继续串行执行 JD 分类、精准评分、STAR 改写和关键词清单生成。
6. 将完整结果原子写入 `result.txt`。
7. 同一次回调将结构化结果渲染到全部结果区域。

网页不复制评分或改写规则，避免网页结果与命令行结果不一致。

## 页面布局

页面标题为“简历 JD 精准匹配 Agent”，包含简短说明和本地文件状态。

输入区采用左右双栏：

- 左栏：可编辑的简历文本框。
- 右栏：可编辑的 JD 文本框。

操作区包含：

- 主按钮“开始精准分析”。
- 次按钮“重新加载文件”。
- 次按钮“保存到文件”。

结果区包含五个标签页：

1. 评分总览：总分、硬性条件分、技术技能分、项目经历分及文字总结。
2. 匹配分析：已匹配技能、缺失技能、不满足的硬性条件。
3. STAR 改写：每个项目的 Situation、Task、Action、Result 和合并文本。
4. 增补关键词：关键词、缺失原因和建议写入位置。
5. 完整结果：标准 JSON。

页面使用少量 Gradio CSS 增强信息层级和响应式显示，不引入外部图片、字体或前端资源。

## 输出结构

分析回调返回稳定的页面视图字典或元组，至少包含：

- 状态信息。
- 评分总览 Markdown。
- 匹配分析 Markdown。
- STAR 改写 Markdown。
- 增补关键词 Markdown。
- 完整 JSON 字典。
- 文字总结。

缺少字段时页面使用安全默认值，不因模型降级警告而崩溃。

## 错误处理

- 页面输入为空：状态区显示错误，不调用 Agent，不写 `result.txt`。
- 输入文件缺失：允许用户在页面中填写；重新加载时显示缺失文件名。
- 输入文件不是 UTF-8 或无法读取：显示读取错误类型，不泄露文件内容。
- 保存输入失败：显示错误，不声明保存成功。
- Agent 返回 `error`：页面显示结构化错误，不展示伪造分数。
- `result.txt` 写入失败：保留页面分析结果并提示导出失败。
- 模型不可用：展示 `warnings`，规则评分、STAR 保守降级和导出继续完成。

服务固定监听 `127.0.0.1:7860`，设置 `share=False`，不创建公网分享链接。

## 测试策略

使用标准库 `unittest` 编写测试，并保持可被 `pytest` 收集。

测试覆盖：

1. 正常读取两个 UTF-8 输入文件。
2. 文件缺失、空内容和解码失败时的页面状态。
3. 显式保存成功并采用 UTF-8。
4. 保存校验失败时不覆盖原文件。
5. 分析使用页面当前文本，且不修改输入文件。
6. 成功分析后生成或更新 `result.txt`。
7. Agent 错误及结果写入错误的页面反馈。
8. 评分、技能、硬性条件、STAR、关键词和 JSON 的视图转换。
9. Gradio 缺失时给出清晰安装提示，而不影响 `src/main.py`。
10. 原有命令行及评分测试全部继续通过。

若当前环境具备 Gradio，则补充执行页面构建冒烟检查；网页逻辑的核心验收不依赖浏览器或公网。

## 验收标准

- `python src/main.py` 的自动文件流程与现有输出保持兼容。
- `python src/web.py` 可在本机启动并自动填充输入框。
- 页面编辑后分析不会覆盖输入文件。
- 显式保存按钮能更新两个输入文件。
- 分析结果与直接调用同一个 Agent 的结果一致，并写入 `result.txt`。
- 页面完整展示用户要求的全部结果类别。
- 服务不监听外部网络接口，不创建共享链接。

## 非目标

- 不发布到公网或 Sites。
- 不增加用户登录、数据库或历史记录。
- 不支持文件上传到外部服务。
- 不改变现有评分权重、技能词库或 STAR 事实安全规则。
